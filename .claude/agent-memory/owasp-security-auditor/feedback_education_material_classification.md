type: feedback
---

claude-nextjs-starterkit 은 공개 학습 자료/스타터킷이다. 다른 개발자가 이 패턴을 자기 production 으로 카피한다는 가정 하에 분류 규칙을 둔다.

**규칙: 직접 익스플로잇 가능 vs 카피-위험만 있는 경우 분리**

- **직접 익스플로잇 가능 (Critical/High/Medium/Low)**: 변경분 자체로 공격자가 무언가 할 수 있는 경우. 이 프로젝트엔 서버/세션/DB 가 없어서 이 카테고리에 해당하는 발견이 매우 드묾.

- **카피-위험만 있는 경우 (Info)**: 학습자가 패턴을 production 으로 옮겼을 때 위험한 경우. 예:
  - 데모 폼의 toast 가 PII 평문 표시 (production 에서 세션 리플레이 캡처 위험)
  - 비밀번호 정책이 `min(8)` 만 (HIBP/zxcvbn 안내 부재)
  - `noValidate` + 클라이언트 검증만 (실제 API 추가 시 서버 재검증 누락 위험)
  - 데모 onSubmit 이 에러 처리 없음 (production 에선 사용자에게 무한 로딩)

Why: 학습 자료에서 약한 패턴을 High 로 부풀리면 진짜 위험에서 주의가 분산된다. 반대로 완전히 무시하면 학습자에게 잘못된 모범을 전파한다. Info 로 기록하되 권고 코드 예시를 함께 제공해 학습 가치를 만든다.

How to apply: 발견 분류 시 자문 — "이 변경분이 머지된 직후 누군가 이 코드베이스를 공격할 수 있는가?" 답이 No 이면 최대 Info. 답이 Yes 이면 그때만 Low 이상.
